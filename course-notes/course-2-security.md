# Course 2: Play It Safe – Manage Security Risks

---

## 📘 Module 1: Security Domains & Risk Management

### CISSP Security Domains

1. **Security & Risk Management**
   - Security posture
   - Risk mitigation
   - Compliance
   - Ethics and regulations  

2. **Asset Security**
   - Data storage and protection
   - Backup and recovery  

3. **Security Architecture & Engineering**
   - Defense in depth
   - Least privilege
   - Zero trust  

4. **Communication & Network Security**
   - Secure network design  
   - Remote access protection  

5. **Identity & Access Management (IAM)**
   - Authentication & authorization  
   - Principle of least privilege  

6. **Security Assessment & Testing**
   - Vulnerability assessments  
   - Security audits  

7. **Security Operations**
   - Incident handling  
   - Monitoring & logging  

8. **Software Development Security**
   - Secure coding practices  
   - Testing & validation  

---

### Key Concepts

- **Risk**: Anything that impacts CIA triad  
- **Security Posture**: Organization’s ability to defend systems  
- **Shared Responsibility**: Everyone plays a role in security

## 📘 Module 2: Frameworks, Controls & Compliance

### NIST Cybersecurity Framework (CSF)

- Identify  
- Protect  
- Detect  
- Respond  
- Recover  

---

### NIST Risk Management Framework (RMF)

1. Prepare  
2. Categorize  
3. Select  
4. Implement  
5. Assess  
6. Authorize  
7. Monitor  

---

### Important Concepts

- **Security Controls**: Safeguards to reduce risk  
- **Compliance**: Following regulations  
- **Security Audit**: Evaluation of controls  

---

## 📘 Module 3: Security Operations & SIEM

### SIEM (Security Information and Event Management)

- Collects log data  
- Detects suspicious activity  
- Generates alerts  
- Helps in incident response  

---

### Tools

- Splunk (Enterprise / Cloud)  
- Google Chronicle  
- Suricata  

---

### Key Concepts

- **Logs**: Records of system activity  
- **IOC (Indicators of Compromise)**: Signs of attack  
- **SOAR**: Automates security responses  

---

## 📘 Module 4: Incident Response

### Incident Response Process

1. Preparation  
2. Detection & Analysis  
3. Containment  
4. Eradication & Recovery  
5. Post-Incident Activity  

---

### Playbooks

- Step-by-step response guides  
- Help standardize incident handling  

---

### Key Example

- **Business Email Compromise (BEC)**  
- Common phishing attack targeting organizations  

---

## 🎯 My Key Learning

- Security is about managing risk, not eliminating it  
- Frameworks like NIST structure security processes  
- SIEM tools are critical for monitoring and detection  
- Incident response requires preparation and documentation  
- IAM and least privilege are essential for preventing breaches
