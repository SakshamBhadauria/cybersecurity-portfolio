# Security Audit Report – Controls & Compliance

## Overview
This security audit was conducted as part of the Google Cybersecurity Professional Certificate.

The objective was to assess security controls and compliance practices for the organization (Botium Toys) and identify potential risks.

---

## Controls Assessment

The following controls were evaluated:

- Least Privilege
- Disaster Recovery Plans
- Password Policies
- Separation of Duties
- Firewall
- Intrusion Detection System (IDS)
- Backups
- Antivirus Software
- Encryption
- Password Management System
- Physical Security Controls (locks, CCTV)
- Fire Detection Systems

### Key Observation
Several critical controls were either missing or not fully implemented, which increases the organization's risk exposure.

---

## Compliance Assessment

### PCI DSS (Payment Card Security)
- Access to credit card data should be restricted
- Secure storage and transmission required
- Encryption must be implemented
- Strong password policies required

---

### GDPR (Data Protection)
- Customer data must be secured
- Breach notification within 72 hours
- Data classification required
- Privacy policies must be enforced

---

### SOC Compliance (System & Organization Controls)
- User access policies required
- Sensitive data must remain confidential
- Data integrity must be maintained
- Authorized access must be ensured

---

## Key Findings

- Weak or missing access control measures
- Lack of encryption for sensitive data
- No proper incident response or recovery planning
- Insufficient monitoring mechanisms

---

## Recommendations

- Implement **least privilege & separation of duties**
- Enable **encryption for sensitive data (PII/SPII)**
- Deploy **IDS/IPS systems**
- Establish **disaster recovery plans**
- Implement **strong password policies + MFA**
- Monitor and phase out legacy systems
- Perform proper **asset classification**
